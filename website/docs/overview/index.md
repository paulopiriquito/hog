# Overview

This section introduces HOG before you configure or extend it. It defines the
vocabulary used throughout the rest of the docs, shows where HOG fits in a
typical frontend/backend stack, and explains what changed in the v2 rewrite.

- [Core concepts](concepts.md) — resources, routes, terminals, the middleware
  chain, and the plugin registry.
- [Use cases](use-cases.md) — the scenarios HOG is built for: SPA hosting, BFF
  authentication, API aggregation, and authorization enforcement.
- [Why v2](why-v2.md) — what changed from the KrakenD-based v1, and where to
  read next.
